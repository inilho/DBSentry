Modules

One tool, the whole pre-flight to post-mortem loop.

Every module below runs against a connection you configure once — no separate credentials to juggle, no separate tool to learn.

Load Test

Custom query slots with bind variables. App-side TPS and p50/p95/p99 latency, not just what the DB reports.

Benchmark Suite

15 locked, repeatable definitions. Warm-up discarded, fixed seed, median/min/max/CV across reps.

Transaction Simulator

Virtual OLTP and OLAP users, each its own session identity — visible in GV$SESSION as real app traffic.

Heavy Jobs

Full scans, hash joins, bulk loads. Every job passes an EXPLAIN PLAN cost gate before it's allowed to run.

Health Check

FRA, TEMP, ASM, RAC, redo, and more at a glance — and a mandatory gate before anything high-risk runs.

Block Check

Surfaces corruption already recorded by RMAN — and says so plainly when nothing has ever been validated.

Live Monitoring

Real-time wait events and active sessions, sampled live — not a cached snapshot from ten minutes ago.

Differential Diagnosis

Ranked hypotheses from wait-event evidence, each with a confidence score and what would refute it.

PM Tools

Parses an AWR Miner file you already have into a dashboard and report. Never queries a licensed view itself.

Reports

Shareable HTML or PDF. Every section states its own limitations instead of quietly hiding them.

On Development

Gather Stats

Schema or full-database statistics gathering, scheduled or on demand, with before/after comparison.

On Development

SQL Advisor

SQL ID-level tuning analysis in the spirit of SQLT — read as clean charts instead of a wall of text.

Guardrails

Every risky action asks permission first.

These aren't settings you have to remember to turn on. They're the default, on every connection, every run.

PROD is hard-blocked

Load tests, benchmarks, heavy jobs, and the transaction simulator all refuse to run against a connection labeled PROD. No override flag.

Writes are opt-in, never silent

Any action capable of DML requires an explicit confirmation. The default is always read-only.

Concurrency is capped

No load test or transaction simulation can spin up unbounded connections against a live instance.

EXPLAIN PLAN before every Heavy Job

A cost-threshold gate runs first. If the plan looks too expensive, the job doesn't start.

License-aware, on purpose

PM Tools never queries a Diagnostics-Pack-licensed view. It only reads a file you already have.

Metrics are labeled honestly

A cumulative counter is never presented as a rate. If a number needs a caveat, the caveat ships with it.
